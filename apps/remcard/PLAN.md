# 记得 Remcard — 产品规划文档

> 状态:🟢在做 · 立项 2026-07-09 · 来源:BACKLOG #2(指数 85)
> 一句话:**离线间隔重复记忆卡——建卡、复习、记住,数据只在这台手机里。**

## 1. 定位与人群

- **解决什么**:背单词/考点/术语靠死记效率低;Anki 强大但界面陈旧、$29.99 劝退新手,云端记忆 app 又要订阅+上传数据。Remcard 用经典 SM-2 间隔重复算法安排复习,纯本地、无账号、无订阅。
- **目标用户**:①备考族(语言/证书/医学术语) ②终身学习者 ③讨厌订阅、注重隐私的工具党。
- **差异化(vs 竞品)**:

| 竞品 | 弱点 | 我们 |
|---|---|---|
| AnkiMobile | 界面陈旧、$29.99 门槛、上手陡 | 现代 Material 3、$4.99 买断、开箱即用 |
| 云端记忆 app(Quizlet 等) | 订阅制、卡片上云 | 一次买断、零网络权限 |
| 系统备忘/笔记 | 无科学复习调度 | SM-2 到期提醒式复习 |

- **卖点文案**:「你的记忆,离线安排」——app 不申请网络权限,商店审核可验证。

## 2. 功能清单

**v1(本次交付,核心闭环)**
- [x] 牌组(Deck):新建/改名/删除;卡片计数 + 今日到期数徽章
- [x] 卡片(Card):正面(问题)/背面(答案)增删改
- [x] 复习会话:按 SM-2 取今日到期卡 → 点卡翻面看答案 → 4 档评分(重来/困难/良好/简单)→ 算法更新间隔与到期日
- [x] 复习进度指示(本轮剩余张数)、无到期卡时空状态引导
- [x] 全本地 JSON 持久化(文档目录),即时保存
- [x] 设置:SM-2 原理说明 + 隐私声明(零网络)
- [x] 免费额度门(免费 2 个牌组,满后引导解锁 Pro;v1 为本地开关,IAP 接入在上架前完成)

**v2(上架后迭代,不阻塞 v1)**
- 备份导出/导入(zip 到分享面板)· 富文本/图片卡面 · 每日提醒通知 · 统计图表(记忆曲线/正确率)· CSV 批量导入 · 桌面小组件今日待复习

**明确不做**:云同步、账号系统、协作、社区共享牌组——违背零服务端定位。

## 3. 页面结构与交互

```
主屏(牌组列表 + 右下「新建牌组」FAB)
 ├─ 点牌组 → 牌组详情(卡片列表 · 顶部「开始复习(N 张到期)」按钮 · FAB 加卡)
 │    ├─ 点卡片 → 卡片编辑(正面/背面)
 │    └─ 长按/滑动 → 删除卡片
 ├─ 复习按钮 → 复习会话(卡面正面 → 点击翻面 → 4 档评分 → 下一张 → 完成)
 └─ 右上设置 → 设置页(壳自带:隐私/关于 + 本 app:SM-2 说明/解锁 Pro)
```

## 4. 技术方案

- **壳**:tool_shell 母版,`RemcardTool implements ToolModule`。
- **算法**:经典 **SM-2**(SuperMemo 2)纯 Dart 确定性实现:`ease`(默认 2.5,下限 1.3)、`interval`、`repetitions`;4 档评分映射 quality。到期日 = 今日 + interval(天)。**有单元测试覆盖调度正确性**。
- **存储**:牌组+卡片序列化为单个 JSON 文件,`path_provider` 取文档目录;千级卡片量内存操作足够。
- **权限**:无。**AndroidManifest 不含 INTERNET**——沿用壳的零网络承诺。
- **依赖**:仅 `path_provider`(存储)。无原生编译、无网络、无三方 SDK ⇒ CI 构建风险极低,本机可 `flutter analyze` + `flutter test` 全验。

**风险与对策**
| 风险 | 对策 |
|---|---|
| SM-2 调度算法写错 | 纯函数 + 单元测试(again 归零重学 / 简单拉长间隔 / ease 下限 1.3) |
| 卡片多时列表卡顿 | ListView.builder 懒构建;千级足够 |
| IAP 未接阻塞收费 | v1 本地闸门+牌组计数,上架前接 in_app_purchase(产品 ID `pro_unlock`) |

- **商店标题**:记得 - 离线间隔重复记忆卡 / Remcard - Offline Spaced Repetition。
