# 回声笔记 EchoJot — 产品规划文档

> 状态:🟢在做 · 立项 2026-07-09 · 来源:BACKLOG #1(指数 88)
> 一句话:**按住说话、松手变文字的离线语音笔记——你的声音不出这台手机。**

## 1. 定位与人群

- **解决什么**:走路/开车/开会时冒出的想法,打字来不及、云端转写 app 要订阅还上传录音。回声笔记按住就录、松手就出文字,全程离线。
- **目标用户**:①随时记想法的创作者/产品人 ②隐私敏感者(律师/医生/记者) ③讨厌订阅制的工具党。
- **差异化(vs 竞品)**:

| 竞品 | 弱点 | 我们 |
|---|---|---|
| Otter/Granola 等云端转写 | 订阅 $10+/月;录音上传云端 | 一次买断;零网络权限 |
| 系统备忘录/录音 | 无转写或需在线;搜不到语音内容 | 离线转写+全文搜索 |
| meetily/FluidVoice(桌面) | 只有桌面端 | 移动端空位正是我们的 |

- **卖点文案**:「你的声音不出这台手机」——app 不申请网络权限,商店审核可验证。

## 2. 功能清单

**v1(本次交付,验收标准=每条都能真机走通)**
- [ ] 点击开录/再点结束(大按钮),录音时波形动画+计时
- [ ] 停止后设备端 whisper 转写(进度指示),自动取首句做标题
- [ ] 笔记时间线:卡片(标题+摘要+时长+日期),下拉刷新无、纯本地即时
- [ ] 全文搜索(标题+正文)
- [ ] 笔记详情:全文可编辑、音频回放、分享/导出文本
- [ ] 删除(滑动/长按)
- [ ] 语言:转写支持中英(whisper 多语模型自动检测);界面中文
- [ ] 免费 30 条,满后弹解锁页(v1 先做本地开关占位,IAP 接入在上架前完成)
- [ ] 设置:模型说明/隐私声明(零网络)/导出全部备份(zip 到分享面板)

**v2(上架后迭代,不阻塞 v1)**
- 逐句时间戳对齐(点句子跳音频)· 实时边说边出字 · 精准大模型可选 · Anki 式待办抽取 · 桌面小组件快捷录音

**明确不做**:云同步、账号系统、协作——违背零服务端定位。

## 3. 页面结构与交互

```
主屏(时间线+搜索框+底部大录音键)
 ├─ 点录音键 → 录音态(波形+计时+停止键) → 停止 → 转写进度 → 新卡片落时间线顶部
 ├─ 点卡片 → 详情(标题/正文可编辑 · 播放条 · 分享 · 删除)
 ├─ 搜索框 → 即时过滤时间线
 └─ 右上设置 → 设置页(壳自带:隐私/关于 + 本 app:模型状态/导出备份/解锁 Pro)
```

## 4. 技术方案

- **壳**:tool_shell 母版,`EchoJotTool implements ToolModule`。
- **录音**:`record` 包,WAV/PCM16/16kHz 单声道(whisper 原生输入格式,免转码)。
- **转写**:`whisper.cpp` 系 Flutter 插件 + **base-q5_1 多语模型(~60MB)打包进 assets**,首启复制到文档目录——不联网下载,保住零网络权限。低端机兜底:转写排队,提示「稍后完成」。
- **存储**:笔记 JSON 索引 + 音频 wav 文件,文档目录;搜索在内存索引上做(千条量级足够)。
- **回放**:`audioplayers`。分享:`share_plus`。
- **权限**:仅 `RECORD_AUDIO`(Android)/ `NSMicrophoneUsageDescription`(iOS)。**manifest 不含 INTERNET**。
- **CI**:模型文件不进 git,workflow 构建前从 HuggingFace 下载到 assets(`prepare_assets.sh`)。

**风险与对策**
| 风险 | 对策 |
|---|---|
| whisper 插件 API 与预期不符 | 本地 flutter analyze 迭代,CI 编译验证 NDK 构建 |
| 低端安卓转写慢 | base→tiny 降级选项;后台队列+通知 |
| 60MB 模型使包体大 | AAB 按需交付上限内(150MB);商店页写明「模型内置,离线可用」把劣势说成卖点 |
| IAP 未接会阻塞收费 | v1 本地闸门+计数,上架前接 in_app_purchase(iOS/Play 各自产品ID `pro_unlock`) |
